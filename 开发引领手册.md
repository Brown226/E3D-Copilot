## 一、核心原则（三句话记住）

1. **PML 优先，C# 为辅** — 复杂查询/批量修改 → 生成 PML 脚本执行；简单属性读写/存在检查 → C# API 直调
2. **安全三层** — 只读工具自动执行、写入操作需确认、批量修改（>5 个）先展示计划再审批
3. **6 + 41 工具架构** — AI 只记住 6 个核心工具名（`query/modify/check/calculate/export/execute_pml`），参数路由到 41 个专用工具

→ 详见 [架构设计](docs/arch/架构设计.md) | [工具设计](docs/design/工具设计.md)

## 二、致命禁区 ❌（写代码前必看）

以下 7 个 API 陷阱是**提前核实发现的致命幻觉**，写错直接编译不通过或运行崩溃：

| ❌ 错误写法 | ✅ 正确写法 | 原因 |
|---|---|---|
| `WindowManager.CreateDockedWindow(k,t,c)` 3 参 | `.Instance.CreateDockedWindow(k,t,c, DockedPosition.Right)` 4 参 | 漏了第 4 参数 `DockedPosition` |
| `CommandLine.Run("$p...")` | `Aveva.Core.Utilities.CommandLine.Command.CreateCommand("...").RunInPdms()` | `CommandLine.Run()` 不存在；命名空间是 `Aveva.Core.*` 非 `Aveva.Pdms.*` |
| `Messaging.WriteInfo(...)` | `Command.CreateCommand("$p ...").RunInPdms()` | `Messaging` 命名空间下无日志静态方法，只有 `PdmsMessage` |
| `elem.GetAttribute("WTHK").ToString()` | `DbElement.GetAsString(DbAttribute.GetDbAttribute("WTHK"))` | 参数是 `DbAttribute` 对象不是字符串；返回值是属性元数据不是值 |
| `Database.GetElement("PIPE-001")` 不存在 | `DbElement.GetElement("PIPE-001")` 存在 | 方法在 `DbElement` 上，不在 `Database` 上 |
| `new ButtonTool(id, title, desc)` | 通过 `CommandManager` 或 `DependencyResolver` 注册 | `ButtonTool` 是抽象类不可实例化 |
| PML 中 `pos.E / pos.N / pos.U` | `!pos.East / !pos.North / !pos.Up` | PML 2 对象成员是全称，不是缩写 |

→ 完整报告详见 [API核实报告](docs/verification/API核实报告.md)

## 三、技术栈红线 🔴

| 约束 | 说明 | 违反后果 |
|------|------|---------|
| .NET Framework **4.8** | 不能用 .NET Core/.NET 5+ 特性 | E3D 不加载 |
| C# **7.3** | 禁止：`IAsyncEnumerable` / `await foreach` / record / target-typed `new()` / 只读结构体 | 编译错误 |
| **流式处理** | 用 `Task` + `Action<Chunk>` 回调，不用 `IAsyncEnumerable` | net48 不支持 |
| **E3D API 线程** | 所有 E3D API 调用必须通过 `ThreadMarshaller` 切换到 UI 线程 | 随机崩溃 |
| **PML 执行入口** | 统一走 `Command.CreateCommand(str).RunInPdms()` | 维护噩梦 |
| **属性读取链路** | `DbElement.GetAsString(DbAttribute.GetDbAttribute("NAME"))` | 返回值错误 |

→ 详见 [源码结构](docs/arch/源码结构.md) §四 命名规范 | [错误处理](docs/arch/错误处理.md) §八 线程模型

### 四项目依赖

```
E3DCopilot.UI (WinForms)  → 依赖 Core + Tools
E3DCopilot.Tools (PML/API) → 依赖 Core
E3DCopilot.Core (逻辑层)   → 无项目依赖
E3DCopilot.Addin (入口)    → 依赖 Core + UI
```

**禁止反向依赖**（Core → Tools → UI）。
