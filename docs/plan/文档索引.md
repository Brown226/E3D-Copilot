# E小智 v1.0 — 总计划

> 本文档是整个 E小智 v1.0 项目的**总领文件**。  
> 每个阶段的具体任务、范围、验收标准在各自的子计划中。
> **先读本文档，再读对应子计划。**

---

## 一、项目愿景

在 AVEVA E3D 工厂设计软件内嵌 AI 编程助手，让工程师用自然语言描述需求，AI 自动生成 PML 脚本并直接执行。

### 一句话定位

> **E3D Copilot — 让 PML 开发效率提升 10 倍**

---

## 二、总范围界定

### ✅ IN SCOPE（v1.0 要做）

| 类别 | 范围 |
|------|------|
| 交互方式 | 嵌入式 DockedWindow + Chat UI |
| AI 后端 | vLLM 本地部署 + Qwen3.5/3.6（OpenAI 兼容） |
| 执行引擎 | PML 脚本执行为主、C# API 直调为辅 |
| 工具数量 | 46 个内置工具（6 核心 + 40 专用） |
| 安全模型 | 三层审批（观察/确认/自动）+ Plan Mode |
| 部署方式 | DotNet DLL 注册（无需 GAC/注册表） |
| 开发环境 | .NET Framework 4.8 + C# 7.3 |
| 长期记忆 | 用户画像 + 自动记忆（Phase 3）|

### ❌ OUT OF SCOPE（v1.0 不做）

| 类别 | 原因 |
|------|------|
| 多 Agent 协作 | 复杂度高，v2 考虑 |
| 多 LLM Provider（除 vLLM）| 团队只用 Qwen |
| Skill 市场/分享 | v2 功能 |
| 移动端/Web 端 | E3D 桌面端独有 |
| E3D 版本适配（非 1.1.x）| 当前只对 E3D 1.1.x |
| i18n 国际化 | 中文独占，v2 考虑 |

## 三、阶段关系图

```
Phase 0: 环境准备（0.5 天）
    │
    ▼
Phase 1: 基础骨架 + MVP（6 天）
    ├── 1a: 最小闭环（D1-D2）
    ├── 1b: MVP 工具（D3-D4）
    ├── 1c: 安全+错误+配置（D5）
    └── 1d: 集成验证+部署（D6）
    │
    ▼
Phase 2: 工具扩展（第 2 周）
    ├── 2a: 管道工具（D8）
    ├── 2b: 设备工具（D9）
    ├── 2c: 批量修改（D10）
    ├── 2d: 检查工具（D11）
    ├── 2e: 导入导出（D12）
    ├── 2f: PML 预览（D13）
    └── 2g: 权限分发（D14）
    │
    ▼
Phase 3: 增强功能（第 3 周）
    ├── 3a: 上下文压缩 + 会话持久化（D15-D16）
    ├── 3b: 长期记忆系统（D16-D18）
    ├── 3c: 安全检查 + 批量限制（D19）
    └── 3d: Skill 系统（D20-D21）
```

### 阶段依赖约束

```
Phase 0 → Phase 1a → Phase 1b → Phase 1c → Phase 1d
                                                   ↓
Phase 2a ─── Phase 2b ─── Phase 2c ─── Phase 2d ──┤
                                                   ↓
                              Phase 2e → Phase 2f → Phase 2g
                                                       ↓
Phase 3a ─── Phase 3b ─── Phase 3c ─── Phase 3d
```

- **禁止跳过 Phase 1a**：最小闭环是所有后续阶段的依赖
- **Phase 2 内部可并行**：如果有多人开发，2a/2b/2d 可并行
- **Phase 3 依赖 Phase 2g**：权限分发完成后才能上线

## 五、跨阶段约束

### 5.1 代码约束

- **技术栈**：.NET 4.8 + C# 7.3（禁止 C# 8+ 特性）
- **API 调用**：所有 E3D API 必须有 `API核实报告.md` 支持
- **PML 调用**：统一走 `Command.CreateCommand(str).RunInPdms()`
- **UI 线程**：所有 E3D API 调用必须通过 `ThreadMarshaller`

### 5.2 文档约束

- 每个阶段结束后，对应的设计文档必须更新到与实现一致
- 新增 API 调用必须补充到 `API核实报告.md`
- `docs/` 中的文档修改必须同步更新 `docs/文档索引.md`

### 5.3 测试约束

- Phase 1a 完成时：必须有 AgentLoop 单元测试
- Phase 1b 完成时：7 个基础工具必须有集成测试
- Phase 1 完成时：整体覆盖率 ≥ 60%
- Phase 2 完成时：整体覆盖率 ≥ 70%

## 六、技术风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 | 负责阶段 |

| .NET 版本限制 | 低 | 中 | 禁止 C# 8+ 特性 | 全阶段 |
| 内存不足（vLLM+E3D） | 中 | 中 | 模型选型 Qwen3.5-32B 可控 | Phase 0 |
| Team 人手不够 | 中 | 高 | Phase 2 内部可并行 | Phase 2 |
