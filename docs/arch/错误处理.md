# E小智 v1.0 — 错误处理与恢复设计

> 设计日期：2026-06-16
> 参考依据：Cline CLI (ClineError 分类 + retry 策略)、Reasonix (checkpoint 快照 + 错误恢复)、MiMo-Code (Effect-TS 错误处理)

---

## 一、设计原则

1. **不抛出未处理异常** — 每一层都有自己的错误处理
2. **LLM 可恢复** — 所有错误以消息形式回传给 LLM，让 AI 自主修复
3. **用户可见** — 错误不沉默，用户能理解"发生了什么"和"怎么解决"
4. **可重试** — 临时故障自动重试，永久故障给出清晰指引
5. **操作可回滚** — 写操作的错误不影响已成功部分

---

## 二、错误分类

| 类别 | 子类 | 说明 | 恢复策略 |
|------|------|------|---------|
| **LLM 错误** | ConnectionError | vLLM 连接失败 | 自动重试 3 次（指数退避） |
| | TimeoutError | LLM 响应超时 | 自动重试 1 次 |
| | ParseError | LLM 返回非 JSON / 无效工具调用 | 自动重发（SanitizeToolPairing） |
| | ContentFilterError | LLM 返回内容被过滤 | 提示用户调整输入 |
| **工具错误** | ToolNotFoundError | 未知工具名 | 重发 LLM（告知可用工具） |
| | InvalidArgsError | 参数格式错误 / 缺少必参 | 重发 LLM（告知正确参数格式） |
| | ToolExecutionError | PML 执行 / C# API 内部错误 | 回滚 + 通知 LLM |
| | PermissionDeniedError | 写操作被拒绝 | 通知用户 |
| **PML 错误** | SyntaxError | PML 脚本语法错误 | LLM 修复脚本重试 |
| | RuntimeError | PML 执行时异常 | LLM 修复脚本重试 |
| | DataError | 元素不存在 / 属性值无效 | 通知 LLM 调整输入 |
| **E3D 错误** | ElementNotFoundError | 目标元素不存在 | 通知 LLM |
| | DbError | 数据库连接 / 事务错误 | 自动重试 1 次 |
| | CommandError | Command.RunInPdms() 失败 | 从 cmd.Error 取 PdmsMessage |
| **系统错误** | ConfigError | 配置加载失败 | 回退默认配置 |
| | MemoryStoreError | SQLite 读写失败 | 降级为内存模式 |
| | ThreadError | 跨线程操作失败 | 自动重试 1 次 |

### 3.1 每一层负责自己的错误

```csharp
// AgentLoop — 顶层错误处理
public async Task<string> RunAsync(CancellationToken ct)
{
    try
    {
        while (!ct.IsCancellationRequested)
        {
            // 1. 调 LLM
            var result = await CallLlmAsync(ct);
            // 2. 解析响应
            var toolCalls = ParseToolCalls(result);
            // 3. 执行工具
            foreach (var call in toolCalls)
            {
                var toolResult = await ExecuteToolAsync(call, ct);
                context.AddResult(call, toolResult);
            }
        }
    }
    catch (CopilotException ex)
    {
        // 已知错误 → 可恢复
        return HandleKnownError(ex);
    }
    catch (Exception ex)
    {
        // 未知错误 → 不可恢复
        return HandleFatalError(ex);
    }
    return null;
}

private string HandleKnownError(CopilotException ex)
{
    switch (ex.ErrorType)
    {
        case ErrorType.LlmConnection:
        case ErrorType.LlmTimeout:
            // LLM 错误 → 重试（自动）
            return RetryStrategy(3, TimeSpan.FromSeconds(1));
        case ErrorType.ToolInvalidArgs:
        case ErrorType.PmlSyntaxError:
            // 工具/PML 语法错误 → 回传 LLM 修复
            return "Error: " + ex.Message + ". Please fix and retry.";
        case ErrorType.PermissionDenied:
            // 权限拒绝 → 通知用户
            sink.Emit(CopilotEvent.ToolResult("操作被拒绝"));
            return "User denied the operation.";
        default:
            return "Error: " + ex.Message;
    }
}

private string HandleFatalError(Exception ex)
{
    // 不可恢复错误 → 记录日志 + 通知用户
    CopilotLogger.Error(ex, "致命错误");
    sink.Emit(CopilotEvent.Error("遇到不可恢复错误，请查看日志"));
    return "Fatal error: " + ex.Message + ". See log for details.";
}
```

## 四、自动重试策略

### 4.1 LLM 调用重试

| 错误类型 | 重试次数 | 退避策略 | 超时 |
|---------|:-------:|---------|:----:|
| ConnectionError | 3 次 | 1s → 2s → 4s（指数退避） | 每尝试 120s |
| TimeoutError | 1 次 | 立即重试（可能是临时加载） | 120s |
| RateLimitError | 3 次 | 10s → 30s → 60s | 无 |

```csharp
public class RetryStrategy
{
    public static async Task<T> WithRetryAsync<T>(
        Func<Task<T>> action,
        int maxRetries,
        TimeSpan baseDelay,
        CancellationToken ct)
    {
        int attempt = 0;
        while (true)
        {
            try
            {
                ct.ThrowIfCancellationRequested();
                return await action();
            }
            catch (Exception ex) when (attempt < maxRetries)
            {
                attempt++;
                long waitMs = (long)(baseDelay.TotalMilliseconds * Math.Pow(2, attempt - 1));
                CopilotLogger.Warn(ex, "重试第 {0} 次，等待 {1}ms", attempt, waitMs);
                await Task.Delay(TimeSpan.FromMilliseconds(waitMs), ct);
            }
        }
    }
}
```

## 五、LLM 输出错误自修复

### 5.1 SanitizeToolPairing

当 LLM 返回格式错误的工具调用时，自动修复再重试：

```csharp
public async Task<string> SanitizeToolPairing(CopilotRequest request)
{
    // 尝试解析 LLM 返回
    var parsed = TryParseToolCalls(request.RawOutput);

    if (parsed == null)
    {
        // 修复常见的格式错误
        request.RawOutput = FixJsonBraces(request.RawOutput);
        request.RawOutput = FixMissingQuotes(request.RawOutput);
        request.RawOutput = FixTrailingComma(request.RawOutput);

        parsed = TryParseToolCalls(request.RawOutput);
    }

    if (parsed == null)
    {
        // 通知 LLM 格式错误
        return "Format error: I could not parse the tool call. "
             + "Please respond with valid JSON matching the tool schema.";
    }

    return parsed;
}
```

### 5.2 常见 LLM 错误模式与修复

| LLM 错误 | 检测 | 自动修复 |
|---------|------|---------|
| 工具名不存在 | 注册表查不到 | 返回"可用工具: [列表]"，让 LLM 重选 |
| 参数名拼写错误 | JSON Schema 校验失败 | 返回"参数 X 无效，有效参数: [列表]" |
| 缺少必参 | JSON Schema 校验失败 | 返回"缺少必参 X"，让 LLM 补全 |
| JSON 格式错误 | JSON 解析失败 | SanitizeToolPairing 自动修复 |
| 调用不存在的工具 | 工具名匹配失败 | 回退到 execute_pml 兜底 |

## 六、PML 执行错误处理

### 6.1 完整的 PML 执行流程

```csharp
public class PmlEngine
{
    public void Run(string pmlScript)
    {
        // PML 脚本自动包裹 Handle 块
        string wrapped = "handle (any) \n"
            + pmlScript + "\n"
            + "endhandle";

        Command cmd = Command.CreateCommand(wrapped);
        bool ok = cmd.RunInPdms();

        if (!ok)
        {
            // 从 Command.Error（PdmsMessage）提取错误信息
            PdmsMessage err = cmd.Error;
            string errCode = err.MessageNumber.ToString();
            string errText = err.MessageText;

            CopilotLogger.Warn("PML 执行错误 [{0}]: {1}", errCode, errText);

            // 常见 PML 错误分类
            switch (errCode)
            {
                case "1001": // 元素不存在
                    throw new CopilotException(ErrorType.ElementNotFound, errText);
                case "2001": // 语法错误
                    throw new CopilotException(ErrorType.PmlSyntaxError, errText);
                default:     // 运行时错误
                    throw new CopilotException(ErrorType.PmlRuntimeError, errText);
            }
        }

        // 返回 cmd.Result（命令行输出）
        return cmd.Result;
    }
}
```

|:---------:|------|------|

| 2xxx | 语法错误 | PML 语法问题 |

## 七、操作回滚

### 7.1 写操作前备份

借鉴 Reasonix 的 Checkpoint 模式：

```csharp
public class ModifyTool : BaseTool
{
    protected override async Task<string> ExecuteInternalAsync(string args)
    {
        // 1. 备份修改前的属性值
        Dictionary<string, string> backup = new Dictionary<string, string>();
        foreach (var (elemName, attrName) in GetChanges(args))
        {
            DbElement elem = NavigateTo(elemName);
            string oldValue = elem.GetAsString(DbAttribute.GetDbAttribute(attrName));
            backup[elemName + "." + attrName] = oldValue;
        }

        try
        {
            // 2. 执行修改
            return await ExecuteModificationsAsync(args);
        }
        catch (Exception ex)
        {
            // 3. 失败 → 回滚
            CopilotLogger.Warn(ex, "修改失败，正在回滚");
            foreach (var (key, oldValue) in backup)
            {
                string[] parts = key.Split('.');
                DbElement elem = NavigateTo(parts[0]);
                DbAttribute attr = DbAttribute.GetDbAttribute(parts[1]);
                elem.SetAttribute(attr, oldValue);
            }
            return "操作失败，已自动回滚。错误: " + ex.Message;
        }
    }
}
```

### 9.2 日志实现

```csharp
public class CopilotLogger
{
    private static readonly string LogDir;
    private static readonly object Lock = new object();

    static CopilotLogger()
    {
        LogDir = Path.Combine(
            Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData),
            "E3DCopilot", "logs");
        Directory.CreateDirectory(LogDir);
    }

    public static void Info(string format, params object[] args)
    {
        WriteLog("INFO", format, args);
        // 可选：通过 PML 输出到 PDMS 消息窗
        // Command.CreateCommand($"$p {string.Format(format, args)}").RunInPdms();
    }

    public static void Warn(Exception ex, string format, params object[] args)
    {
        WriteLog("WARN", format + " | {1}", args, ex.Message);
    }

    public static void Error(Exception ex, string format, params object[] args)
    {
        WriteLog("ERROR", format + " | {1}", args, ex.ToString());
    }

    private static void WriteLog(string level, string format, params object[] args)
    {
        string line = string.Format("{0:yyyy-MM-dd HH:mm:ss.fff} [{1}] {2}",
            DateTime.Now, level, string.Format(format, args));
        lock (Lock)
        {
            string file = Path.Combine(LogDir,
                "e3dcopilot." + DateTime.Now.ToString("yyyy-MM-dd") + ".log");
            File.AppendAllText(file, line + Environment.NewLine);
        }

        // 也输出到 Debug 窗口（开发时可见）
        System.Diagnostics.Debug.WriteLine(line);
    }
}
```
