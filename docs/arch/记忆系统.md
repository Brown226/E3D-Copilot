# E小智 v1.0 — 长期记忆系统

> 设计日期：2026-06-16  
> 设计依据：对比 Reasonix (Auto-Memory)、Cline CLI (Rule + Context)、MiMo-Code (SQLite FTS5 + /learn)  
> 设计目标：让用户越用越懂我 — 记住偏好、记住项目、记住纠正

---

## 一、四层记忆架构

```
                    ┌─────────────────────────┐
                    │  ④ 项目知识库           │ ← 项目特定规范/约定
                    │  AGENTS.md (多级)        │   用户可编辑
                    │  项目规则 (条件式)       │
                    ├─────────────────────────┤
                    │  ③ 自动记忆 (SQLite)    │ ← AI 主动管理
                    │  remember / forget /     │   全局 + 项目 双范围
                    │  memory (BM25 搜索)      │   索引注入 System Prompt
                    ├─────────────────────────┤
                    │  ② 用户画像 (JSON)      │ ← 自动学习
                    │  偏好 / 习惯 / 风格      │   每次工具调用更新
                    │  反馈学习闭环            │
                    ├─────────────────────────┤
                    │  ① 会话历史 (JSONL)     │ ← 持久化 + 可检索
                    │  对话记录 / 操作日志     │   回收站 / 归档 / 压缩
                    └─────────────────────────┘
```

| 层级 | 存储位置 | 持久性 | 更新方式 | 访问范围 |
|------|---------|--------|---------|---------|
| ① 会话历史 | `sessions/` JSONL | 永久（可删） | 自动追加 | 检索注入新会话 |
| ② 用户画像 | `user_profile.json` | 永久 | 自动学习 | 每次对话注入 prompt |
| ③ 自动记忆 | `memory/` SQLite FTS5 | 永久 | AI 工具主动管理 | 每次对话检索注入 |
| ④ 项目知识库 | `AGENTS.md` + `project_knowledge.json` | 永久 | 用户/半自动 | 读取即注入 |

## 三、第 ① 层：会话历史

### 3.1 设计目标

- 每次会话结束后自动持久化，跨会话可检索
- 注入最近 N 次会话摘要到新会话，让 AI "记得上次干啥了"
- 支持按日期/关键词/元素名搜索历史

### 3.2 存储格式

**会话索引**（`sessions/index.json`）：
```json
[
  {
    "id": "sess_20260616_001",
    "date": "2026-06-16T14:30:00Z",
    "title": "修改 ZONE-PIPE-01 管道壁厚",
    "summary": "查询 50 根 DN100 管道，修改 WTHK 从 SCH30→SCH40",
    "tools_used": ["query", "modify", "check"],
    "element_count": 50,
    "duration_ms": 45000,
    "status": "completed"
  }
]
```

**会话 JSONL**（`sessions/2026-06/sess_001.jsonl`）：
```jsonl
{"role":"user","content":"把 ZONE-PIPE-01 所有 DN100 的 WTHK 改成 SCH40","timestamp":"..."}
{"role":"assistant","content":"已修改 38 根管道","timestamp":"...","tools":[{"name":"modify","args":{...}}]}
{"role":"user","content":"不对，只改 ZONE-01 的","timestamp":"..."}
```

### 3.3 注入策略

新会话开始时，注入最近 3 次会话摘要：

```
最近操作记录：
- 2026-06-16 14:30 修改 ZONE-PIPE-01 的 38 根管道壁厚从 SCH30→SCH40
- 2026-06-16 10:00 查询 ZONE-PIPE-01 所有 DN100 管道，列出 DIA/WTHK/SPEC
- 2026-06-15 16:00 导出 ZONE-STRU-01 所有梁的尺寸到 Excel
```

### 3.4 回收站与归档

- 删除的会话移入 `.trash/`，30 天后自动清理
- 可设置归档阈值（会话超过 1000 条则归档旧会话）

## 四、第 ② 层：用户画像

### 4.1 设计目标

自动学习用户操作习惯，越用越懂用户。无需用户手动配置。

### 4.2 数据结构

```json
{
  "id": "user_default",
  "version": 1,
  "created": "2026-06-16T00:00:00Z",
  "updated": "2026-06-16T14:30:00Z",
  "project": "ZONE-PIPE-01",

  "preferences": {
    "element_types": {
      "PIPE": 120,
      "EQUI": 45,
      "STRU": 20,
      "BRAN": 10
    },
    "attributes": {
      "DIA": 95,
      "WTHK": 88,
      "SPEC": 60,
      "NAME": 45,
      "ROOM_NO": 30
    },
    "scopes": {
      "ZONE-PIPE-01": 95,
      "ZONE-EQUI-01": 40,
      "ZONE-STRU-01": 15
    },
    "output_format": "table",
    "output_language": "zh-CN",
    "verbosity": "concise"
  },

  "common_values": {
    "pipe_spec": ["CS300", "SS316"],
    "wall_thickness": ["SCH40", "SCH30"],
    "pipe_diameter": ["DN100", "DN150", "DN200"],
    "fluid": ["WATER", "STEAM"]
  },

  "naming_patterns": {
    "pipe": "PIPE-{zone}-{num}",
    "equipment": "EQUI-{type}-{num}",
    "nozzle": "NOZZ-{parent}-{size}"
  },

  "safety": {
    "preferred_level": "confirm_batch",
    "auto_approve_readonly": true,
    "confirm_batch_threshold": 10
  },

  "behavior": {
    "active_hours": [8, 18],
    "common_tasks": {
      "morning": ["query", "check"],
      "afternoon": ["modify", "export"],
      "end_of_day": ["export", "check"]
    }
  }
}
```

### 4.3 自动学习规则

每次工具调用后，根据参数更新画像：

```
query(type="PIPE")           → element_types.PIPE += 1
query(attributes=["WTHK"])   → attributes.WTHK += 1
query(scope="ZONE-PIPE-01")  → scopes.ZONE-PIPE-01 += 1
modify(attr="WTHK", val="SCH40") → common_values.wall_thickness.SCH40 += 1
modify(scope="ZONE-PIPE-01")  → scopes.ZONE-PIPE-01 += 1
```

用户纠正行为：

```
用户: "不对，我说的是 ZONE-01 里的"
  → scopes.ZONE-PIPE-01 += 5（纠正权重更高）

用户: "不用导出了，用表格显示就行"
  → output_format = "table"

用户: "以后这种操作直接执行，不用确认"
  → safety.preferred_level = "auto"

用户: "这个项目里管道都叫 PIPE-XX-XXX"
  → naming_patterns.pipe = "PIPE-{num}"
```

### 4.4 注入 System Prompt

```
用户偏好：常用元素 PIPE/EQUI，常用属性 DIA/WTHK/SPEC，常用区域 ZONE-PIPE-01
  常用等级 CS300/SS316，常用壁厚 SCH40/SCH30，常用管径 DN100/DN150/DN200
  输出偏好：表格形式
  安全偏好：只读工具自动执行，批量修改（≥10条）需确认
```

### 4.5 权重衰减

画像权重会随时间衰减（近期操作比远期操作更重要）：

```
weight = base_weight × 0.95^(days_since_last_use)
```

确保画像反映的是用户"当前"习惯，而非"历史"习惯。

## 五、第 ③ 层：自动记忆（借鉴 Reasonix）

### 5.1 设计目标

AI 主动管理持久记忆，不是被动记录对话。借鉴 Reasonix 的 `remember/forget/memory` 三工具模式。

### 5.2 存储

- 存储：SQLite + FTS5 全文搜索（借鉴 MiMo-Code）
- 分类：4 种类型（user / feedback / project / reference）
- 范围：全局（跨项目）+ 项目（仅限当前项目）
- 索引：`MEMORY_文档索引.md` 注入 System Prompt

### 5.3 三工具设计

#### remember 工具

```
用途：保存值得长期记住的事实
类型：user / feedback / project / reference
输入：name（slug）、title、description（一行钩子）、body（详情）
规则：
  - 只记非显而易见的事实
  - feedback/project 类型必须包含 "Why:" 和 "How to apply:" 行
  - 用 [[name]] 链接相关记忆
  - 保存前先 search 查重，复用已有 name 来更新而非新建
```

#### forget 工具

```
用途：删除不再适用的记忆
输入：name（slug）
效果：立即从索引中移除
```

#### memory 工具

```
用途：搜索/读取/列出记忆
操作：search（BM25 搜索）/ read（读全文）/ list（列索引）
参数：query / name / type / limit（默认 8，最大 20）
输出：带 score 和 snippet，与 Reasonix 相同的格式
```

### 5.4 记忆类型说明

| 类型 | 内容 | 示例 | 范围 |
|------|------|------|------|
| **user** | 用户是谁、有什么偏好、擅长什么 | "用户是管道专业工程师，常用 PML 写批量修改脚本" | 全局 |
| **feedback** | 工作方式指导（Why + How） | "管道壁厚修改时，优先检查 FLUID 是否为 WATER" | 项目 |
| **project** | 项目约束、目标、约定 | "管道命名：PIPE-{zone}-{num}，所有管道必须有 FLUID 属性" | 项目 |
| **reference** | 外部资源指针 | "E3D API 文档路径: ...\E3D官方API文档\docs\" | 项目 |

### 5.5 索引注入

每次对话开始时，`MEMORY_文档索引.md` 被注入 System Prompt prefix：

```
Saved memories in <project>/memory/:
- [常用管径](common-pipe-diameters.md) type=user - 常用管径: DN100/DN150/DN200
- [管道命名规范](naming-convention.md) type=project - PIPE-{zone}-{num} 格式
- [壁厚修改注意事项](wall-thickness-notes.md) type=feedback - 改前检查 FLUID
```

LLM 看到摘要后，按需 `memory(operation="read", name="...")` 读取全文。

## 七、反馈学习闭环

### 7.1 完整流程

```
用户: "把所有 DN100 管道壁厚改成 SCH40"
  → AI: modify(type="PIPE", attr="WTHK", value="SCH40", filter="*DN100*")
  → 结果: 修改了 50 根管道
  
用户: "不对，只改 ZONE-PIPE-01 里的"      ← 纠正
  → AI 学习:
    1. scopes.ZONE-PIPE-01 += 5 (纠正权重高)
    2. 记录: "用户习惯在 ZONE-PIPE-01 操作"
    3. 下次类似请求自动限定 scope
  
用户: "以后用表格显示就行"               ← 纠正
  → AI 学习:
    1. output_format = "table"
    2. 后续所有导出默认表格格式
  
用户: "这个项目管道都叫 PIPE-XX-XXX"     ← 纠正
  → AI 学习:
    1. naming_patterns.pipe = "PIPE-{num}"
    2. 写入 project_knowledge.json
```

### 7.2 反馈类型与存储

| 用户反馈 | 学习内容 | 存储位置 |
|---------|---------|---------|
| "不对，是 XXX 不是 YYY" | 纠正值偏好 | 用户画像 → common_values |
| "只改 ZONE-01 里的" | 范围偏好 | 用户画像 → scopes |
| "用表格显示" | 输出格式偏好 | 用户画像 → output_format |
| "以后直接执行，不用确认" | 安全偏好 | 用户画像 → safety.preferred_level |
| "这个项目管道叫 PIPE-XX-XXX" | 命名规范 | project_knowledge.json |

### 7.3 权重机制

纠正的权重 > 正常操作 > 猜测：

```
正常操作: weight += 1
纠正行为: weight += 5
用户明示: weight += 10
```

确保纠正能覆盖之前的猜测。

## 八、自动记忆建议（借鉴 Reasonix）

### 8.1 Memory Suggestion 机制

定期扫描历史会话，提取值得记住的内容，推荐给用户：

```
触发时机：
1. 用户说 "记住这个" / "以后..." / "始终..."
2. AI 发现重复模式（同一个操作出现 ≥3 次）
3. 每次会话结束时，AI 检查是否有值得记录的内容

提取示例：
  "用户在过去 5 次对话中 3 次查询 ZONE-PIPE-01 的 DN100 管道"
  → 建议: "保存 '常用查询 ZONE-PIPE-01 DN100 管道' 到记忆?"
  
  "用户多次修改管道壁厚，每次都忘记指定范围"
  → 建议: "保存 '修改管道壁厚时必须指定 scope' 作为反馈?"
```

### 8.2 工作流识别

检测重复操作序列，生成工作流建议：

```
检测到用户连续执行:
  1. query(type="PIPE", filter="ZONE-PIPE-01")
  2. check(type="attribute_complete", scope="ZONE-PIPE-01")
  3. modify(attr="ROOM_NO", value="RM-001", scope="ZONE-PIPE-01")
  
建议: "创建一个 'ZONE-PIPE-01 属性补全' 工作流 (Skill)?"
```

## 九、技术实现要点

### 9.1 SQLite FTS5（借鉴 MiMo-Code）

```csharp
// 使用 Microsoft.Data.Sqlite（.NET 4.8 可用）
CREATE VIRTUAL TABLE memories USING fts5(
    name, title, description, body, type, scope
);

// BM25 搜索（原生支持）
SELECT name, title, description, snippet,
       bm25(memories) AS score
FROM memories
WHERE memories MATCH ?
ORDER BY score
LIMIT ?;
```

### 9.2 权重衰减

```csharp
// 画像权重衰减（每天衰减 5%）
public double DecayWeight(double weight, int daysAgo)
{
    return weight * Math.Pow(0.95, daysAgo);
}
```
