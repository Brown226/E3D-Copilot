## 四、通信协议 (前端 ↔ C# 后端)

### 4.1 消息格式

所有通信使用 JSON over `WebView2.PostWebMessageAsString()`:

```json
{
  "type": "消息类型",
  "payload": { ... }
}
```

### 4.2 前端 → C# 消息

| type | payload | 说明 |
|------|---------|------|
| `user:message` | `{ text: string }` | 用户发送消息 |
| `user:cancel` | `{}` | 取消当前处理 |
| `user:new_session` | `{}` | 新建会话 |
| `user:approve` | `{ toolId: string, allow: bool }` | 审批工具执行 |
| `user:settings` | `{ ... }` | 保存设置 |
| `user:export` | `{ format: string }` | 导出会话 |

### 4.3 C# → 前端 消息

| type | payload | 说明 |
|------|---------|------|
| `host:ready` | `{ version: string }` | 宿主就绪 |
| `llm:stream:delta` | `{ delta: string }` | 流式 token |
| `llm:stream:end` | `{ usage: ... }` | 流结束 |
| `llm:thinking` | `{ text: string }` | 思考过程 |
| `tool:dispatch` | `{ id, name, args }` | 工具触发 |
| `tool:result` | `{ id, result, error? }` | 工具完成 |
| `tool:approval` | `{ id, name, args, description }` | 请求审批 |
| `error` | `{ message: string }` | 错误通知 |
| `state:update` | `{ ... }` | 状态更新 |
| `history:list` | `{ sessions: [...] }` | 历史会话列表 |

### 4.4 通信时序

```
User: 输入 "查 DN100 管道"
  │
  ├→ JS: bridge.send('user:message', { text: "查 DN100 管道" })
  │
  ├→ C#: Controller.SendAsync() → AgentLoop.Start()
  │
  ├→ C# ← vLLM: SSE streaming (token by token)
  │    │
  │    ├→ JS: { type: 'llm:thinking', payload: { text: "..." } }
  │    │     (可选：AI 展示推理过程)
  │    │
  │    ├→ JS: { type: 'llm:stream:delta', payload: { delta: "正在查" } }
  │    ├→ JS: { type: 'llm:stream:delta', payload: { delta: "询DN" } }
  │    ├→ JS: { type: 'llm:stream:delta', payload: { delta: "100..." } }
  │    │
  │    ├→ C#: LLM 返回 tool_call → ToolRegistry.Execute()
  │    │    │
  │    │    ├→ JS: { type: 'tool:dispatch', payload: { id, name, args } }
  │    │    ├→ C#: DbElement.GetElement() + 查询属性
  │    │    └→ JS: { type: 'tool:result', payload: { id, result } }
  │    │
  │    ├→ C# ← vLLM: 继续 streaming (基于 tool result)
  │    │
  │    └→ JS: { type: 'llm:stream:delta', payload: { delta: "找到..." } }
  │
  └→ JS: { type: 'llm:stream:end', payload: { usage: {...} } }
```

## 五、C# 后端关键接口

### IE3DTool — 工具统一接口

```csharp
public interface IE3DTool
{
    string Name { get; }
    string Description { get; }
    ToolCapability Capability { get; }  // Read | Write | Execute
    Task<ToolResult> ExecuteAsync(ToolContext ctx);
}

public class ToolContext
{
    public CopilotSession Session { get; init; }
    public JsonElement Args { get; init; }
    public CancellationToken Cancellation { get; init; }
}

public class ToolResult
{
    public bool Success { get; init; }
    public string? Error { get; init; }
    public object? Data { get; init; }
}
```

### CopilotController — 中央调度

```csharp
public class CopilotController
{
    // 通信入口（由 WebView2 桥接调用）
    public async Task HandleMessageAsync(BridgeMessage msg);

    // Agent 循环
    public async Task SendAsync(string input);

    // 审批
    public void Approve(string toolId, bool allow);

    // 会话
    public void NewSession();
}
```

### Bridge — WebView2 IPC 桥接

```csharp
public class Bridge : IDisposable
{
    private readonly WebView2 _webView;
    private readonly CopilotController _controller;
    
    // 接收前端消息
    private void OnWebMessageReceived(object sender, WebView2WebMessageReceivedEventArgs e);
    
    // 发送到前端
    public void SendToFrontend(string type, object? payload);
}
```

## 八、构建与部署

### 开发流程

```
# 1. 启动前端 HMR 服务器
cd web-ui && npm run dev
→ http://localhost:5173

# 2. 在 E3D 中加载 Addin (或使用 TestHost)
#    开发模式下，WebView2 指向 localhost:5173
#    代码修改即时生效，无需重新编译 C#

# 3. 构建部署
cd web-ui && npm run build:addin
→ 输出到 src/E3DCopilot.WebHost/wwwroot/

# 4. 编译 C#
msbuild E3DCopilot.sln
```

### 两个模式

| 模式 | WebView2 URL | 适用 |
|------|-------------|------|
| **开发** | `http://localhost:5173` | 前端开发，HMR 热更新 |
| **生产** | `file:///.../wwwroot/index.html` | 内网部署，零网络依赖 |

### 部署产物

```
E3DCopilot.Addin.dll     # 插件入口
E3DCopilot.Core.dll       # 核心逻辑
E3DCopilot.Tools.dll      # 工具
E3DCopilot.WebHost.dll    # WebView2 宿主（含 wwwroot/）
```

部署方式：将 DLL 放入 E3D 插件目录，在 `DesignAddins.xml` 中注册。

## 十、关键决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
