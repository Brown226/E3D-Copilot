## 五、依赖关系

```
e3d-ui (React SPA, 构建产物 → WebHost/wwwroot)
     ↑ 构建输出
     |
E3DCopilot.WebHost (WebView2 宿主, GUI)
     ↓
E3DCopilot.Tools (PML + E3D API 桥接)
     ↓
E3DCopilot.Core (纯逻辑，通过接口与 E3D 解耦)
     ↓
E3DCopilot.Addin (E3D 插件入口)
```

- Core 通过 `IToolDispatcher` / `IE3DEnvironment` 接口与 Tools 解耦
- Tools 的 `RealE3DEnvironment` 依赖真实 `Aveva.*` DLL
- Tools 的 `SimulatedE3DEnvironment` 用于离线开发/测试
- Addin 和 Loader 都引用 WebHost + Core + Tools

## 六、命名规范

| 项 | 规范 | 示例 |
|----|------|------|
| 项目名称 | `E3DCopilot.{Area}` | `E3DCopilot.Core` |
| 命名空间 | `E3DCopilot.{Area}.{Sub}` | `E3DCopilot.Core.Tools.Handlers` |
| 类名 | PascalCase | `AgentLoop` |
| 接口 | `I` 前缀 | `ICopilotProvider` |
| 文件命名 | 与类名一致 | `AgentLoop.cs` |
| 测试文件 | `{ClassName}Tests.cs` | `ToolExecutorTests.cs` |
