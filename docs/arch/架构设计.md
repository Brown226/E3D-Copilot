### 2.3 AgentLoop (执行层)

**职责**: 核心 Agent 循环，LLM 调用 + 工具执行

**核心循环**:
```csharp
public async Task RunAsync(CancellationToken ct, string input)
{
    session.Add(new Message(Role.User, input));

    for (int step = 0; step < maxSteps; step++)
    {
        // 1. 调用 LLM 流式完成
        var (text, reasoning, calls, usage) = await StreamAsync(ct);

        // 2. 保存 assistant 消息
        session.Add(new Message(Role.Assistant, text, calls));

        // 3. 无工具调用 → 检查最终答案质量
        if (calls.Count == 0)
        {
            if (FinalReadinessCheck()) return;
            continue; // 质量不达标，重试
        }

        // 4. 执行工具调用
        var results = await ExecuteBatchAsync(calls);

        // 5. 保存工具结果
        session.AddToolResults(results);

        // 6. 上下文压缩（可选）
        maybeCompact();
    }
}
```

**流式 Chunk 处理**:
```csharp
private async Task<(string text, string reasoning, List<ToolCall> calls, Usage usage)>
    StreamAsync(CancellationToken ct)
{
    string text = "";
    string reasoning = "";
    var calls = new List<ToolCall>();
    Usage usage = null;

    // net48：通过回调接收流式 Chunk（不支持 IAsyncEnumerable）
    await provider.StreamAsync(request, (chunk) =>
    {
        switch (chunk.Type)
        {
            case ChunkType.Reasoning:
                reasoning += chunk.Text;
                sink.Emit(new CopilotEvent { Kind = EventKind.Reasoning, Text = chunk.Text });
                break;
            case ChunkType.Text:
                text += chunk.Text;
                sink.Emit(new CopilotEvent { Kind = EventKind.Text, Text = chunk.Text });
                break;
            case ChunkType.ToolCall:
                calls.Add(chunk.ToolCall);
                sink.Emit(new CopilotEvent { Kind = EventKind.ToolDispatch, Tool = chunk.ToolCall });
                break;
            case ChunkType.Usage:
                usage = chunk.UsageData;
                break;
        }
    }, ct);

    return (text, reasoning, calls, usage);
}
```

**工具并行分区** (借鉴 Reasonix):
```csharp
private async Task<List<ToolResult>> ExecuteBatchAsync(List<ToolCall> calls)
{
    var results = new List<ToolResult>();

    // 分区: 连续 ReadOnly 并行, Writer 串行
    var batches = PartitionToolCalls(calls);

    foreach (var batch in batches)
    {
        if (batch.All(c => tools.Get(c.Name).IsReadOnly))
        {
            // 并行执行
            var tasks = batch.Select(c => ExecuteOneAsync(c));
            results.AddRange(await Task.WhenAll(tasks));
        }
        else
        {
            // 串行执行
            foreach (var call in batch)
            {
                results.Add(await ExecuteOneAsync(call));
            }
        }
    }

    return results;
}
```

**单个工具执行** (含权限检查):
```csharp
private async Task<ToolResult> ExecuteOneAsync(ToolCall call)
{
    var tool = tools.Get(call.Name);

    // 1. Plan Mode 检查
    if (planMode && !tool.IsReadOnly)
        return ToolResult.Blocked("Plan mode: write operations disabled");

    // 2. 权限检查
    var approval = await policy.CheckAsync(tool, call.Args);
    if (approval == ApprovalResult.NeedApproval)
    {
        var req = new PendingApproval { Tool = tool, Args = call.Args };
        approvals[req.Id] = req;
        sink.Emit(new CopilotEvent { Kind = EventKind.ApprovalRequest, Approval = req });

        // 阻塞等待用户审批
        var result = await req.WaitAsync();
        if (!result.Allow) return ToolResult.Rejected("User rejected");
    }

    // 3. 执行
    try
    {
        var output = await tool.ExecuteAsync(call.Args);
        sink.Emit(new CopilotEvent { Kind = EventKind.ToolResult, Output = output });
        return ToolResult.Success(output);
    }
    catch (Exception ex)
    {
        return ToolResult.Error(ex.Message);
    }
}
```

### 2.5 PML Execution Engine (执行层)

**职责**: PML 脚本生成与执行，是工具调用的核心执行层

**设计决策**: PML 为主，C# API 为辅

| 层 | 技术 | 适用场景 |
|----|------|---------|
| 工具接口层 | C# `IE3DTool` | 统一注册、权限控制、事件分发 |
| 执行引擎层 | **PML 脚本** | 复杂查询/修改/批量操作（核心） |
| 底层桥接 | `Command.CreateCommand(str).RunInPdms()` | PML 与 C# 的桥梁（`Aveva.Pdms.Utilities.CommandLine`）|
| 简单操作 | C# API 直调 | 单个属性读写、元素存在性检查 |

**PML 黄金范式** (90%+ 工具的核心骨架):
```
1. 构建集合:  var !coll append coll all TYPE with Matchwild(name,'*PATTERN*')
2. 遍历处理:  DO !val values !coll → $!val 导航 → 属性读写
3. 存在验证:  var !flag exist $!name → TRUEA/FALSEA 分支
4. 外部交互:  import 'PMLFileBrowser' 导入 → writefile 导出
5. 异常处理:  handle (code) / handle any → 跳过/记录/提示
```

**PML 几何操作速查**（PML POSITION/DIRECTION 对象，注意成员是 East/North/Up 不是 E/N/U）:
```pml
!pos = !!ce.Position             -- 读取元素位置
!pos.East = 1000                 -- 东坐标（不是 .E）
!pos.North = 2000                -- 北坐标（不是 .N）
!pos.Up = 3000                   -- 高坐标（不是 .U）
!dist = !posA.Distance(!posB)    -- 两点距离
!mid = !posA.Mid(!posB)          -- 中点
!dir = !posA.Direction(!posB)    -- 方向向量
```

**核心组件**:

    {

## 四、LLM Provider

**vLLM + Qwen 适配** (OpenAI 兼容接口，net48 版):

> ⚠️ net48 不支持 `IAsyncEnumerable<T>` / `await foreach`。流式处理改为 `Task` + `StreamReader` 逐行回调。

```csharp
public class VllmProvider : ICopilotProvider
{
    private readonly HttpClient http;
    private readonly string baseUrl = "http://localhost:8000/v1";  // vLLM 默认端口
    private readonly string model = "Qwen3.6-72B";  // 或 Qwen3.5-32B

    // net48 流式方案：传入回调，内部逐行调用
    public async Task StreamAsync(
        CopilotRequest request,
        Action<Chunk> onChunk,
        CancellationToken ct)
    {
        var body = new
        {
            model = request.Model ?? model,
            messages = request.Messages.Select(m => new { role = m.Role, content = m.Content }),
            tools = request.Tools.Select(t => new { /* JSON Schema */ }),
            stream = true
        };

        // vLLM 兼容 OpenAI 格式: /v1/chat/completions
        var response = await http.PostAsJsonAsync($"{baseUrl}/chat/completions", body, ct);
        var stream = await response.Content.ReadAsStreamAsync(ct);

        using (var reader = new StreamReader(stream))
        {
            while (!reader.EndOfStream)
            {
                ct.ThrowIfCancellationRequested();
                var line = await reader.ReadLineAsync();
                if (string.IsNullOrEmpty(line)) continue;

                var json = JsonSerializer.Deserialize<VllmChunk>(line);

                if (json.Choices != null && json.Choices.Length > 0
                    && json.Choices[0].Delta != null)
                {
                    var delta = json.Choices[0].Delta;

                    if (delta.ToolCalls != null)
                    {
                        foreach (var tc in delta.ToolCalls)
                            onChunk(Chunk.ToolCall(tc.Function.Name, tc.Function.Arguments));
                    }
                    else if (delta.Content != null)
                    {
                        onChunk(Chunk.Text(delta.Content));
                    }
                }

                if (json.Usage != null)
                    onChunk(Chunk.Usage(json.Usage.CompletionTokens, json.Usage.PromptTokens));
            }
        }
    }
}
```

## 八、线程模型

**核心约束**：E3D API 要求在 E3D 主线程（UI 线程）上调用。

```
UI 线程 (E3D 主线程)
  ├── WebViewForm (WinForms UserControl + WebView2)
  ├── E3D API 调用 (DbElement, CommandLine, WindowManager)
  └── Bridge 消息接收 (OnWebMessageReceived)

后台线程 (Task/ThreadPool)
  ├── AgentLoop.RunAsync (LLM 调用 + 循环控制)
  ├── VllmProvider.StreamAsync (HTTP 流式)
  ├── Bridge.SendToFrontend → PostWebMessageAsString (线程安全)
  └── Tool.ExecuteAsync → 需切换到 UI 线程调用 E3D API
```

**线程切换机制**：
```csharp
public class ThreadMarshaller
{
    private readonly SynchronizationContext uiContext;

    public ThreadMarshaller()
    {
        // 在 E3D 主线程初始化时捕获
        uiContext = SynchronizationContext.Current;
    }

    // 在 UI 线程执行（E3D API 调用必须走这里）
    public Task<T> InvokeOnUIThread<T>(Func<T> action)
    {
        var tcs = new TaskCompletionSource<T>();
        uiContext.Post(_ =>
        {
            try { tcs.SetResult(action()); }
            catch (Exception ex) { tcs.SetException(ex); }
        }, null);
        return tcs.Task;
    }

    // 在 UI 线程执行（无返回值）
    public Task InvokeOnUIThread(Action action)
    {
        var tcs = new TaskCompletionSource<object>();
        uiContext.Post(_ =>
        {
            try { action(); tcs.SetResult(null); }
            catch (Exception ex) { tcs.SetException(ex); }
        }, null);
        return tcs.Task;
    }
}
```

## 九、审批流同步机制

**问题**：AgentLoop 在后台线程运行，审批需要 UI 线程弹出对话框，用户确认后继续。

**解决方案**：`TaskCompletionSource` + `SynchronizationContext`

```csharp
public class PendingApproval
{
    public string Id { get; } = Guid.NewGuid().ToString("N");
    public IE3DTool Tool { get; set; }
    public string Args { get; set; }
    public string Description { get; set; }

    // 内部完成源
    private readonly TaskCompletionSource<ApprovalResult> tcs
        = new TaskCompletionSource<ApprovalResult>();

    // UI 线程调用：用户点击审批按钮
    public void Complete(bool allow, bool sessionPersist = false)
    {
        tcs.SetResult(new ApprovalResult
        {
            Allow = allow,
            SessionPersist = sessionPersist
        });
    }

    // 后台线程调用：阻塞等待审批结果
    public Task<ApprovalResult> WaitAsync()
    {
        return tcs.Task;
    }
}

// AgentLoop 中的审批流程
private async Task<ToolResult> ExecuteOneAsync(ToolCall call)
{
    var tool = tools.Get(call.Name);
    var approval = await policy.CheckAsync(tool, call.Args);

    if (approval == ApprovalResult.NeedApproval)
    {
        var req = new PendingApproval { Tool = tool, Args = call.Args };

        // 发送审批事件到 UI 线程（非阻塞）
        sink.Emit(new CopilotEvent
        {
            Kind = EventKind.ApprovalRequest,
            Approval = req
        });

        // 后台线程阻塞等待用户操作（不占用 UI 线程）
        var result = await req.WaitAsync();
        if (!result.Allow) return ToolResult.Rejected("User rejected");

        // 会话级持久化
        if (result.SessionPersist)
            policy.Set(call.Name, ApprovalMode.Auto);
    }

    // 切换到 UI 线程执行 E3D API
    var output = await tool.ExecuteAsync(call.Args);
    return ToolResult.Success(output);
}
```

**审批状态流转**：
```
AgentLoop (后台线程)
  → sink.Emit(ApprovalRequest)  ──── 非阻塞发送
  → await req.WaitAsync()       ──── 阻塞等待
                                      ↑
Bridge (WebView2)                       |
  → SendToFrontend("tool:approval")     |
  → React ApprovalDialog 展示          |
  → 用户点击 批准/拒绝            ──── 用户操作
  → Bridge 收到 user:approve           |
  → req.Complete(allow)          ──── 唤醒后台线程
```

## 十、安全模型

**三层安全**:
```
1. Tool Policy — 静态规则（per-tool auto/ask/planOnly）
2. Permission Gate — 运行时检查（敏感操作需确认）
3. Code Review — 生成 PML 脚本预览（用户审批后执行）
```

**安全分级**:

| 级别 | 工具类型 | 默认行为 | 示例 |
|------|---------|---------|------|
| **L0 只读** | 查询/读取 | 自动执行 | query_elements, get_attributes |
| **L1 导航** | 视图切换 | 自动执行 | navigate_to, get_hierarchy |
| **L2 写入** | 单个修改 | 需确认 | set_attribute, modify_pipe_spec |
| **L3 批量** | 批量修改 | 需确认 + 预览 | batch_set_attribute |
| **L4 系统** | PML/Undo | 需确认 + 预览 | execute_pml, undo |

**E3D 特殊安全考虑**:
- E3D 进程内执行，无沙箱 → 必须代码级审查
- 写操作影响整个设计模型 → 需要预览 + 确认
- 批量操作风险高 → 限制单次操作数量
- 操作可逆性 → 记录操作日志，支持撤销
