# Markdown 渲染测试清单

> 对 E小智 说以下每条提示词，检查渲染结果。✅=正常 ❌=异常

---

## 1. 基础表格（多行）
**提示词：**
```
用表格列出 E3D 中最常用的 5 个 query 参数，包含参数名、类型、说明三列
```

**检查点：**
- [ ] 表头 + 分隔行 + 数据行正确渲染为 HTML 表格
- [ ] 列对齐正常

---

## 2. 内联表格（单行 || 分隔 — 重点）
**提示词：**
```
用表格简要列出 E3D 的几个设计专业：设备、结构、管道、暖通、支吊架，每个一行，格式紧凑
```

**检查点：**
- [ ] LLM 可能输出单行 `||` 分隔 —— 应自动拆分为多行表格 ✅
- [ ] 表格正常渲染，不显示原始 `|` 字符

---

## 3. 表格中包含特殊字符
**提示词：**
```
用表格列出 3 个 PML 函数及其示例，函数名用反引号包裹
```

**检查点：**
- [ ] 表格内反引号代码正常渲染
- [ ] 表格线不被破坏

---

## 4. 代码块（围栏）
**提示词：**
```
写一段 PML 代码示例，展示如何查询当前元素的名称和类型
```

**检查点：**
- [ ] 代码块有语法高亮
- [ ] 有语言标签显示
- [ ] 复制按钮出现

---

## 5. 内联代码
**提示词：**


**检查点：**
- [ ] 内联代码有灰色背景高亮
- [ ] 不会和普通文字粘连
